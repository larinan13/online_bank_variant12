# Отчёт по практике (вариант 12)

## 1. Тема
Онлайн-запись в банк: вклады, консультации, контроль кредитной истории.

## 2. Сущности
- employees (сотрудники: должность, отдел)
- clients (клиенты, паспорт, возраст 18+)
- operations (типы операций: вклад/консультация)
- credit_histories (кредитный рейтинг, флаг плохой истории)
- bookings (запись: дата, время, статус, сумма)

## 3. Ограничения
- UNIQUE (email, passport, time slot employee+date+time)
- CHECK (возраст 18+, credit_score 300-850)
- FOREIGN KEY (RESTRICT для bookings, CASCADE для credit_histories)
- Триггер: блокировка крупного вклада (>=1 млн) при is_bad_history=TRUE

## 4. Нормализация (3NF)
- Нет повторяющихся групп (1НФ)
- Нет частичных зависимостей (2НФ)
- Нет транзитивных зависимостей (3НФ) – сумма вклада только в bookings

## 5. Индексы
- idx_booking_date, idx_status, idx_department, idx_credit_bad

## 6. Запросы
1. Ранжирование сотрудников по числу уникальных клиентов за месяц (RANK)
2. Консультации с HAVING и LEFT JOIN
3. Доля сотрудника в операциях отдела (оконная SUM)

## 7. Тесты (выполнены)
- Дублирование времени записи → ошибка
- Клиент 17 лет → ошибка CHECK
- Удаление сотрудника с записями → RESTRICT
- Клиент с плохой историей + вклад 1.5 млн → триггер блокирует
- Успешный вклад с хорошей историей → проходит

## 8. Вывод
Спроектирована и реализована БД для банка. Выполнена проверка кредитной истории. Создан отчётный запрос с оконной функцией.
